# 2FA / Two-Factor Authentication

Achylo uses **TOTP (Time-based One-Time Password)** with Google Authenticator (or any compatible app like Authy) to protect sensitive actions such as withdrawals.

---

## Setting Up 2FA

1. Go to your **Profile** → **Security** section.
2. Click **Enable 2FA**.
3. A QR code and a manual secret key will be shown.
4. Scan the QR code with **Google Authenticator** or enter the secret key manually.
5. Enter the **6-digit code** shown in the app and click **Activate 2FA**.
6. **Save your backup codes immediately**. They will not be shown again.

---

## Authenticator Label

The entry in your authenticator app will be labeled as:

```
Achylo Pay: 0xYourWalletAddress
```

The `Achylo Pay` prefix is the issuer, and the wallet address is the account identifier.

---

## Backup Codes

Backup codes are **one-time emergency codes** generated by Achylo. They let you complete sensitive actions when you **cannot access your authenticator app**.

### When to use them
- You lost your phone.
- You uninstalled or reset Google Authenticator.
- Your phone is damaged, stolen, or unavailable.

### Format
Backup codes look like:

```
A3F2-9B1C
8D1E-4F7A
```

Each code is **8 characters** separated by a hyphen.

### How to use them
When a screen asks for your **2FA code**, enter a backup code instead of the 6-digit authenticator code. Each backup code **works only once** and is then marked as used.

### Where they work
- Withdrawal validation (`/api/v1/security/2fa/validate-withdrawal`)
- Disabling 2FA (`/api/v1/security/2fa/disable`)
- Regenerating backup codes (`/api/v1/security/2fa/regenerate-backup-codes`)

### Important notes
- Backup codes are shown **only once** — when you first enable 2FA or after regenerating them.
- You receive **10 backup codes** per set.
- Store them **offline** in a safe place (paper, password manager, encrypted file).
- If you run out of backup codes, use the **Regenerate** button (requires your current authenticator code or a backup code).

---

## Regenerate Backup Codes

The **Regenerate** button creates a brand new set of 10 backup codes and **invalidates the previous set**. This is useful if:

- You think someone saw your backup codes.
- You already used several of them.
- You want to rotate them as a security measure.
- You lost your phone and need a fresh set of backup codes.

You must provide your current **6-digit authenticator code** or a **backup code** to regenerate.

---

## Disabling 2FA

To disable 2FA:

1. Enter your current **6-digit authenticator code** or a **backup code** in the **Disable 2FA** section.
2. Click **Disable 2FA**.

This will:
- Remove the TOTP secret from your account.
- Delete all backup codes.
- Turn off 2FA protection.

> **Warning:** Disabling 2FA reduces your account security. You will no longer need a 2FA code for withdrawals.

---

## Recovery Flow (Lost Phone)

If you lose your phone but still have your backup codes:

1. Go to **Profile → Security**.
2. In the **Disable 2FA** section, enter a **backup code** instead of the authenticator code.
3. Click **Disable 2FA**.
4. 2FA is now disabled and all old backup codes are deleted.
5. Re-enable 2FA on your new phone by scanning the new QR code.
6. Save the new backup codes immediately.

## FAQ

### What code do I enter to disable 2FA?
Either your **6-digit authenticator code** or a **backup code** (format `XXXX-XXXX`).

### What is the difference between the secret key and backup codes?
- **Secret key**: Used by Google Authenticator to generate 6-digit codes every 30 seconds.
- **Backup codes**: One-time emergency codes generated by Achylo for when you cannot access your authenticator app.

### Can I use backup codes to disable 2FA?
Yes. Backup codes work for disabling 2FA and regenerating backup codes.

### What happens if I lose both my phone and backup codes?
Contact Achylo support. You will need to verify ownership of your wallet through a signed message or other identity verification.

## Related API Endpoints

| Endpoint | Description |
|----------|-------------|
| `POST /api/v1/security/2fa/setup` | Generate QR code and secret |
| `POST /api/v1/security/2fa/verify` | Activate 2FA with first code |
| `GET /api/v1/security/2fa/status` | Check if 2FA is enabled |
| `POST /api/v1/security/2fa/disable` | Disable 2FA (requires current code) |
| `POST /api/v1/security/2fa/validate-withdrawal` | Validate withdrawal (TOTP or backup code) |
| `POST /api/v1/security/2fa/regenerate-backup-codes` | Generate new backup codes |
